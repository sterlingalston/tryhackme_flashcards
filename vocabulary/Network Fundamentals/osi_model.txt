4→Encapsulation|The process of adding layer-specific information to data as it travels down the OSI model stack before transmission.|OSI model

7→Layer 2 - Data Link|OSI layer responsible for physical addressing (MAC addresses) and presenting data in a format suitable for transmission.|OSI layer
8→Layer 3 - Network|OSI layer responsible for routing and reassembling data packets; uses IP addresses; includes protocols like OSPF and RIP.|OSI layer

10→Layer 4 - Transport|OSI layer that determines how data is transmitted between devices; uses TCP or UDP.|OSI layer
11→Layer 5 - Session|OSI layer that creates, maintains, and terminates connections (sessions) between devices; supports checkpointing to resume transmission after a dropped connection.|OSI layer
12→Layer 6 - Presentation|OSI layer that standardises and translates data formats between the application layer and lower layers; acts as a data translator so different applications display data consistently.|OSI layer
13→Layer 7 - Application|OSI layer where user-facing protocols operate; provides GUIs and rules for how users interact with data sent or received; includes DNS.|OSI layer

15→NIC|Network Interface Card; hardware in every network-enabled computer that comes with a unique MAC address assigned by the manufacturer.|networking hardware
16→OSPF|Open Shortest Path First; a Layer 3 routing protocol that determines optimal data paths based on fewest hops and reliability.|routing protocol
17→RIP|Routing Information Protocol; a Layer 3 routing protocol used to determine optimal paths for data across a network.|routing protocol
18→TCP|Transmission Control Protocol; a Layer 4 protocol that guarantees data delivery through error-checking and maintains a constant connection.|protocol
