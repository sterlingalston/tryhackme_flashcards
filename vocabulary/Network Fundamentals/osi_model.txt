5→Layer 2 - Data Link|OSI layer responsible for physical addressing (MAC addresses) and presenting data in a format suitable for transmission.|OSI layer
6→Layer 3 - Network|OSI layer responsible for routing and reassembling data packets; uses IP addresses; includes protocols like OSPF and RIP.|OSI layer|https://sterlingalston.github.io/tryhackme/networking%20fundamentals/.attachments/236265ca1f6b3f9a11052c714701b16ca5d0a273.png
7→Layer 4 - Transport|OSI layer that determines how data is transmitted between devices; uses TCP or UDP.|OSI layer
8→Layer 5 - Session|OSI layer that creates, maintains, and terminates connections (sessions) between devices; supports checkpointing to resume transmission after a dropped connection.|OSI layer
9→Layer 6 - Presentation|OSI layer that standardises and translates data formats between the application layer and lower layers; acts as a data translator so different applications display data consistently.|OSI layer
10→Layer 7 - Application|OSI layer where user-facing protocols operate; provides GUIs and rules for how users interact with data sent or received; includes DNS.|OSI layer|https://sterlingalston.github.io/tryhackme/networking%20fundamentals/.attachments/2f937e841f48fb259be97717b3ea09eedb3b92a3.png

13→RIP|Routing Information Protocol; a Layer 3 routing protocol used to determine optimal paths for data across a network.|routing protocol
14→TCP|Transmission Control Protocol; a Layer 4 protocol that guarantees data delivery through error-checking and maintains a constant connection.|protocol|https://sterlingalston.github.io/tryhackme/networking%20fundamentals/.attachments/330022b5ee40a2547be2ef394e3f0cc53988c388.png
15→UDP|User Datagram Protocol; a Layer 4 protocol that sends data without guaranteeing delivery; faster but less reliable than TCP.|protocol|https://sterlingalston.github.io/tryhackme/networking%20fundamentals/.attachments/98bc7f637f6168915754eaec239fad5c23de432f.png
