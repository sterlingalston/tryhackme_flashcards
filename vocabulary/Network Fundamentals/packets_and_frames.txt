1→Packet|A Layer 3 (Network Layer) data unit containing an IP header and payload; the basic unit of data transmission across networks.|networking concept
2→Frame|A Layer 2 (Data Link) data unit that encapsulates a packet and adds MAC addresses; does not contain an IP address.|networking concept
3→Encapsulation|The process of wrapping data with layer-specific headers as it travels down the OSI model stack; each layer adds its own information.|networking concept
4→TTL|Time to Live; a field in a packet header that sets an expiry timer to prevent packets from clogging a network if they never reach their destination.|packet header
5→Checksum|A packet header field used for integrity checking; if data is altered in transit, the checksum will not match and the packet is considered corrupt.|packet header
6→Source Address|The IP address of the sending device stored in a packet header, so data knows where to return to.|packet header
7→Destination Address|The IP address of the receiving device stored in a packet header, so data knows where to travel next.|packet header
8→TCP/IP Model|A 4-layer networking model (Application, Transport, Internet, Network Interface); a summarised version of the OSI model; connection-based and uses encapsulation.|networking model
9→Source Port|A TCP header field containing the randomly-chosen port number opened by the sender to send a TCP packet (range 0-65535).|TCP header
10→Destination Port|A TCP header field containing the fixed port number of the application or service on the receiving host (e.g. port 80 for a webserver); not chosen at random.|TCP header
11→Sequence Number|A TCP header field with a random number assigned to the first piece of transmitted data; used to reconstruct data in the correct order.|TCP header
12→Acknowledgement Number|A TCP header field containing the next expected sequence number (previous sequence number + 1); confirms receipt of data.|TCP header
13→Flag|A TCP header field that determines how a packet is handled during the handshake process (e.g. SYN, ACK, FIN, RST).|TCP header
14→SYN|The initial TCP handshake packet sent by a client to initiate a connection and synchronise sequence numbers with the server.|TCP handshake
15→SYN/ACK|The second TCP handshake packet sent by the server to acknowledge the client's SYN and provide its own Initial Sequence Number.|TCP handshake
16→ACK|An acknowledgement TCP packet used by either device to confirm that a series of messages/packets have been successfully received.|TCP handshake
17→DATA|A TCP message type used to transmit actual data (bytes of a file) once a three-way handshake connection has been established.|TCP handshake
18→FIN|A TCP packet used to cleanly close an established connection after data transfer is complete.|TCP handshake
19→RST|A TCP packet that abruptly terminates all communication; used as a last resort when a problem occurs (e.g. service not working, low resources).|TCP handshake
20→ISN|Initial Sequence Number; the random number assigned to the first data packet in a TCP connection; each subsequent packet increments by 1.|TCP concept
